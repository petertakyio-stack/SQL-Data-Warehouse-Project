# SQL-Data-Warehouse-Project

Welcome to my first tutorial project **Data Warehouse and Analysis Project** repository.

This project basically demonstrates my understanding in building a very comprehensive data warehousing and analytics solution, straight from builidng a data warehouse to generating meaningful and actionable business insights to inform decision making. This project is designed as a portfolio project and it highlights industry best practicies in data engineering and analytics using SQL. 

---

**Project Requirements**

### Building the Data Warehouse (Data Engineering)

### Objective
To develop a modern data warehouse using SQL Server to consolidate sales data, enbling analytical reporting and informed decision-making.

### Project Specifications
1. **Data Source**: Data will be imported from two sources (ERP and CRM) provided in a CSV file
2. **Data Quality**: Data quality issues will be cleansed and resolved prior to the analysis phase
3. **Integration**: Combine both sources into a single, user-friendly data model designed for analytical queries.
4. **Scope**: Focus on the latest dataset only; historisation of data is not required
5. **Documentation**: Provide clear documentation of he data model to support both business stakeholders and analytics teams.

---

### BI: Analytics & Reporting (Data Analysis)

### Objective
Develop SQL-based analytics to deliver detailed insights into
1. **Customer Behaviour**
2. **Product Performance**
3. **Sales Trend**

These insights will empower stakeholders with key business metrics, enabling strategic decision-making.
